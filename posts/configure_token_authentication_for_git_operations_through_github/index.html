Configure token authentication for git operations through GitHub#

GitHub has announced token authentication requirements for git operations . They will not support password authentication for git operations and will require using personal access tokens over HTTPS or SSH keys by August 13, 2021.

The recommendation is to use personal access tokens and it is explained in this youtube video about personal access tokens .

Setting up personal access tokens#

You can generate the personal access token in GitHub by going into >Settings >Developer settings >Personal access tokens

When you are prompted for your credentials during a git operation, use your personal access token instead of your password. Your personal access token will only be visible on GitHub the first time it’s generated, you will not be able to see it once you leave the page after you’ve generated it.

You might want to cache it on your computer so you don’t get prompted for your your email and password, or in this case your personal access token, each time. Type in the follow command in your git project directory to cache your login credentials:

#need to type this before git asks for your login credentials usually during a push or pull request
git config --global credential.helper cache

Once you’ve initiated caching, start a pull or push request. For the password field, copy and paste in the token. You shouldn’t need to type your credentials in the next time you push or pull. If for some reason your computer prompts for a username and password again you can just re-generate the token or create a new token. You can change your token at any time by re-generating or deleting and creating a new one.

You can create multiple personal access tokens and configure them with different permissions and use/associate them for different operations. I’ve only needed the repo permissions thus far but you have the option to enable others:

Commands and files related to this topic that might be useful:

# see the file the stores your git configurations:
cat ~/.gitconfig

# edit the file the stores your git configurations (can use either vim, nano or other terminal editors):
vim ~/.gitconfig

#sets your git profile's email address your user.email without going into .gitconfig
git config --global user.email "example@email.com"

#shows what your git profile's user.name and user.email variables are set to
git config -l

##deletes the cache for your credentials
git config --global --unset credential.helper

Setting up SSH authentication#

If you don’t have a ssh key or you don’t know if you do follow the guide in Generating a new ssh key and adding it to the ssh agent .

To generate a ssh key for github use the following command and replace the example email with your GitHub email.

ssh-keygen -t ed25519 -C "your_email@example.com"

If you think you might have a ssh key, you can check the id_rsa.pub file (default location of ssh keys):

cat ~/.ssh/id_rsa.pub

Or just check the ssh directory to see what files there are

ls ~/.ssh

Then follow the guide in Adding a new ssh key ot your github account to add the SSH key to your GitHub account.
